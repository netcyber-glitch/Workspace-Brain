# Workspace Brain 구축 계획서

## 1. 개요 (Overview)
* **목표**: MRA, AI_Music_Hub, Where_AI, Rental 등 여러 프로젝트에 산재한 개발 문서, 접근 기록, 기획서 등을 한 곳에서 수집, 인덱싱하여 자연어 검색 및 파일 간 연관 관계 추적이 가능한 통합 지식 베이스(개인화된 AI 검색 엔진)를 로컬 독립형 프로그램으로 구축합니다.
* **형태**: Python 기반의 독립형 (Standalone) 프로그램 (향후 `.exe` 형태 패키징 지원).
* **기대 효과**: 과거 문서 파편화로 인한 컨텍스트 유실 방지, 자연어 질의를 통한 신속한 히스토리 파악, 파일 간 참조 관계를 통한 구조적 이해도 상승.

---

## 2. 필수 기능 정의 (Core Features)

### 2.1 문서 수집 및 파싱 (Data Ingestion)
* **다중 디렉토리 스캔**: 지정된 여러 루트 폴더를 재귀적으로 스캔합니다.
* **지원 포맷 파싱**: `.md`, `.txt`, `.json` 등 주요 텍스트 기반 문서를 파싱합니다.
* **메타데이터 추출**: 파일 생성일, 최종 수정일, 파일 크기, 절대 경로, 문서 제목(Heading 1 기반)을 추출합니다.
* **연관 관계(Edge) 추출**: 문서 내 포함된 마크다운 링크(예: `[참고](파일명.md)`), 동일 폴더 내 인접성, 혹은 문서 내 특정 토큰(타 프로젝트명 언급 등)을 기반으로 문서 간의 논리적 연결 고리를 분석합니다.

### 2.2 인덱싱 및 하이브리드 저장소 (Indexing & Storage)
* **청킹(Chunking)**: 긴 문서를 의미 단위(문단 또는 특정 토큰 길이 기준)로 분할하여 검색 정확도를 높입니다.
* **벡터 데이터베이스**: 분할된 텍스트를 임베딩(Embedding) 벡터로 변환하여 저장합니다. 자연어 문맥 검색(Semantic Search)을 가능하게 합니다.
* **관계 메타데이터 관리**: 파일 각각을 노드(Node)로, 파일 간 관계를 엣지(Edge)로 다루어 관계형 DB나 경량화된 그래프 데이터로 관리합니다.

### 2.3 지능형 검색 및 응답 엔진 (Search & RAG)
* **자연어 검색**: "MRA와 Where AI 간의 인증 로직에 대한 문서를 찾아줘"와 같은 사용자 질의를 이해하고 관련 문서를 반환합니다.
* **메타데이터 필터링**: 질의어 외에도 특정 날짜 구간, 프로젝트 폴더 단위, 특정 파일 확장자만 좁혀서 검색하는 하이브리드 필터링을 지원합니다.
* **AI 요약 응답 (RAG)**: 검색된 상위 3~5개의 문서 컨텍스트를 종합하여, 단순 문서 목록 나열에 그치지 않고 AI가 직접 요약된 답변을 생성합니다.

## 3. 사용자 인터페이스 (UI) 설계

이 시스템은 개발 및 기획의 생산성을 높이기 위해 직관적이고 깔끔한 UI를 제공해야 합니다.

### 3.1 메인 대시보드 (Dashboard)
* **통합 검색 바**: 화면 중앙 상단에 위치한 넓은 자연어 검색 입력창.
* **인덱싱 현황 요약**: 스캔된 전체 문서 수, 프로젝트별 문서 비율, 최근 업데이트된 폴더 등의 통계를 위젯 형태로 제공.
* **최근 활동**: 가장 최근에 생성/수정된 문서 목록 (Quick Access).

### 3.2 검색 결과 뷰 (Search Results)
* **AI 인사이트 패널**: 검색 결과 최상단에 검색된 문서들을 바탕으로 AI가 요약 해답을 제시하는 영역.
* **결과 카드 리스트**: 문서 제목, 소속 프로젝트(태그 부착), 관련 내용 스니펫(검색어 하이라이팅), 생성 시간, 연관 파일 개수가 포함된 리스트.

### 3.3 지식 그래프 맵 (Knowledge Graph Explorer)
* **시각화 캔버스**: 문서들을 점(Node)으로, 연관 관계를 선(Edge)으로 표현하는 인터랙티브 네트워크 그래프.
* **상세 정보 사이드바**: 그래프에서 특정 노드를 클릭하면 우측에서 슬라이드 아웃되어 해당 문서의 요약 내용과 메타데이터를 보여줍니다.

### 3.4 설정 패널 (Settings)
* **경로 관리 (Workspace Paths)**: 감시 및 인덱싱할 대상 폴더(`d:\MRA`, `d:\AI_Music_Hub` 등)를 추가/삭제하는 UI.
* **동기화 옵션 (Sync Options)**: 수동 '지금 인덱싱 갱신' 버튼 및 주기적(예: 하루 1회) 자동 갱신 스케줄 설정.
* **AI 모델/API 설정**: 로컬 LLM을 사용할지, 클라우드 API(Gemini/OpenAI 등)를 사용할지 선택하고 필요한 키를 입력하는 보안 설정 창.

## 4. 아키텍처 및 추천 기술 스택

| 구분 | 추천 프레임워크/라이브러리 | 사유 |
| :--- | :--- | :--- |
| **언어** | Python 3.10+ | AI, 데이터 파싱, 로컬 DB 생태계가 가장 압도적으로 풍부함. |
| **UI 프레임워크** | Streamlit (또는 PyQt6/PySide6) | Streamlit은 로컬 웹 기반으로 데이터 앱을 극도로 빠르게 제작가능. 독립 윈도우 창이 필수라면 PyQt 사용. |
| **AI 연동 파이프라인**| LangChain | 다양한 문서 포맷 로더(Loader)와 텍스트 분할기(Splitter) 기본 내장. |
| **벡터 데이터베이스** | ChromaDB (로컬) | 별도의 서버 설치 없이 Python 내장 형태로 로컬 파일 시스템에 벡터 DB 구축 가능. |
| **그래프 시각화** | pyvis (HTML/JS 기반) | Python에서 네트워크 그래프를 그리면 브라우저에서 인터랙티브하게 조작 가능한 HTML 생성. |
| **실행 파일 패키징** | PyInstaller | 완성된 Python 스크립트와 의존성 패키지를 하나의 윈도우 `.exe` 파일로 압축/빌드 가능. |

## 5. 단계별 개발 계획 (Phases)

*   **Phase 1: Foundation (기반 및 수집)**
    *   `d:\Workspace_Brain` 구조 셋업.
    *   지정된 대상 폴더들을 읽어 파일 정보와 텍스트를 재귀적으로 수집하는 파서 모듈 개발.
*   **Phase 2: Brain Mapping (인덱싱 및 시각화 대비)**
    *   텍스트 청킹, 임베딩 모델 연결, ChromaDB를 이용한 로컬 벡터 인덱싱 로직 구현.
    *   문서 간 참조 관계 추출 로직 작성.
*   **Phase 3: Search Engine & UI (검색 API 및 화면)**
    *   자연어 질의를 벡터로 변환하여 매칭 문서를 찾아내는 검색 모듈 통합.
    *   Streamlit(또는 선택된 프레임워크)을 이용해 대시보드 및 결과 레이아웃 구축.
*   **Phase 4: Refinement & Packaging (안정화 및 .exe 빌드)**
    *   검색 품질 튜닝(프롬프트 조정, 필터 성능 개선).
    *   PyInstaller를 적용하여 별도의 파이썬 설치 없이도 타겟 PC에서 단독 실행 가능한 실행 파일 제작 테스트.
