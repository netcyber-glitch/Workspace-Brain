# 기존 상용 LLM 웹 브라우저 연동 대체 방안 (API 미사용)

현재 상용 웹 버전(ChatGPT Plus, Claude Pro 등)을 이미 결제해서 쓰고 있는데, 추가적인 API 비용을 지출하지 않고 (그리고 로컬 구축 전 단계에서) **어떻게 하면 'Workspace Brain' 앱의 검색 결과를 내가 쓰는 기존 LLM 브라우저 창으로 버튼 하나로 넘길 수 있는지**에 대한 구체적인 구현 방안입니다.

완벽한 백그라운드 무인(無人) 봇 처리는 상용 서비스 약관상 API 없이 차단되지만, 사용자의 복잡한 "문서 찾기 -> 여러 개 긁어서 모으기 -> 복사 -> 브라우저 띄우기 -> 붙여넣기"를 클릭 한 번으로 끝내주는 **스마트 매크로(RPA) 연동**은 아주 훌륭한 대안입니다.

---

## 1. 클립보드 매크로 & 브라우저 포커싱 (가장 추천)

이 방식이 복사-붙여넣기의 귀찮음을 99% 없애주는 실질적인 최선책이자 안전한 방법입니다.

### 📝 작동 흐름
1. **검색 및 조립**: 사용자가 Workspace Brain에서 "I18n 설계 문서 찾아줘"라고 검색합니다. 프로그램이 관련 문서 3~5개를 찾아냅니다. 
2. **원클릭 버튼**: 결과 창 아래에 있는 **`[ 🚀 내 ChatGPT 창으로 질문 넘기기 ]`** 버튼을 누릅니다.
3. **자동 텍스트 조립**: 프로그램이 백그라운드에서 지시문과 3개 문서를 합쳐서 1개의 완벽한 프롬프트 문자열로 만듭니다. (예: *"다음 문서를 바탕으로 내 질문에 대답해. [문서1 전문]..."*)
4. **강제 클립보드 복사**: 만들어진 초대용량 텍스트를 프로그램이 윈도우 **클립보드에 조용히 복사(`Ctrl+C`)** 해 버립니다. 
5. **브라우저 호출 및 자동 붙여넣기**:
   - 프로그램이 사용자가 이미 띄워놓은 'ChatGPT 화면(크롬/엣지 등)' 창을 찾아내어 윈도우 화면 최상단으로 **강제 팝업 포커스**를 맞춥니다.
   - 키보드 매크로(예: 파이썬의 `pyautogui` 라이브러리)를 쏴서, 입력창에 **자동으로 `Ctrl+V` 와 `Enter` 키를 입력**합니다.

### 👍 장점
- 개발 난이도가 낮고 윈도우 OS 위에서 안정적으로 돌아갑니다.
- 브라우저 제재(로봇 캡차 봇 차단 등) 위험이 0%에 수렴합니다. 사람이 브라우저에 클립보드 붙여넣기를 한 것과 완벽히 똑같이 인식하기 때문입니다.
- 단일 줄넘김이나 URL의 길이 제한 없이 수만 자의 긴 문서 묶음도 문제없이 한 방에 넘길 수 있습니다.

## 2. URL Query (파라미터 전달) 방식 (단순하지만 제약 많음)

크롬을 호출할 때 주소창 뒤에 내가 물어볼 말을 꼬리표(`?q=...`)로 달아서 창을 열어버리는 방식입니다.

- `https://chatgpt.com/?q=[조립된 텍스트 전체]` 구조로 파이썬 모듈(`webbrowser`)이 실행합니다.
- **장점**: 개발 코드가 길어봤자 3줄이고 윈도우 포커싱 에러에서 자유롭습니다.
- **단점**: 이게 작동하려면 주소 표시줄(URL)에 수천 자의 텍스트가 들어가게 되는데 대부분의 브라우저가 이걸 중간에 끊어 먹습니다(보통 2,000~8,000자 초과 시 잘림). 결국 문서 1~2개 이상을 못 넘길 가능성이 큽니다.

## 3. 웹훅 (Webhook) + 브라우저 확장(Extension) 방식 (복잡함)

사용자의 크롬 브라우저에 'Workspace Brain 수신용 확장 프로그램'을 하나 깝니다.

1. Workspace Brain(파이썬 앱)이 검색된 문서 뭉치를, 윈도우 내부에 떠 있는 로고 포트(예: 내 컴퓨터 안의 `localhost:8080`)로 조용히 쏩니다.
2. 크롬 확장 프로그램이 이 신호를 낚아채서, 내가 띄워놓은 ChatGPT 탭을 감지하여 입력칸(textarea) DOM 객체 안에 자바스크립트로 텍스트를 직접 꽂아 넣고, `Submit` 버튼을 프로그래밍적으로 누릅니다.

- **장점**: 마우스/키보드가 막 움직이는 형태가 아니므로 훨씬 우아하고 프로그래밍적(현대적)입니다.
- **단점**: 파이썬 데스크톱 프로그램 외에 크롬 확장팩까지(JS, Manifest) 만들어야 하므로 유지보수가 피곤하고 타겟 사이트(ChatGPT, Claude 등) 디자인이 바뀔 때마다 확장 프로그램이 종종 뻗습니다.

---

## 결론: 그래서 어떻게 만들어야 하는가?

**초기 모델은 1번(클립보드 매크로 & 창 포커스) 방식을 도입하는 것이 가장 가성비가 좋고 정신 건강에 이롭습니다.**

사용자 관점에서 볼 때:
API 유료 결제를 단돈 1원도 내지 않고 기존에 돈 내고 쓰는 상용 웹 창을 그대로 활용하면서 버튼 클릭 한 번이면 "Workspace Brain이 알아서 똑똑하게 문서를 긁어모음 -> 상용 웹 입력창으로 자동 전송 및 타격 발사(Enter)"라는 아름다운 콤보를 만들어낼 수 있습니다. 

수동으로 폴더 열어서 그 안의 내용 드래그-복사하고 다시 브라우저 돌아와서 붙여넣는 막노동을 완벽히 퇴출시키는 "RPA 방식"이 현재로선 상용 LLM의 폐쇄성(API 강요 정책)을 뚫는 최선의 정답입니다.
